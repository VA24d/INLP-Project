The Memorization Crisis

Large Language Models gain their intelligence by processing billions of internet tokens — which means they also store exact copies of copyrighted books, private data, and sensitive content. A model can recite verbatim passages from Harry Potter simply because the text appeared in its training corpus. This creates serious legal and ethical risks.

The complete solution would be to delete the data and retrain from scratch, but for frontier models like Gemma or Llama that is far too expensive. Instead, Machine Unlearning seeks to surgically modify a pre-trained model so it forgets specific information while keeping its general capabilities intact.

This project investigates a critical and largely overlooked gap: most evaluations test high-precision (FP16) models, but real deployments always compress models to 8-bit or 4-bit. Does the forgotten knowledge stay forgotten after quantization? We introduce the Recovery Delta (ΔRec) metric to measure exactly that.

📚

Forget Target

All seven Harry Potter books (~1.1M tokens) — verbatim passages, character knowledge, spells, and lore deeply embedded in the model's weights.

🛡️

Retain Target

Harry Potter FanWiki articles (~500K tokens) — general language ability the model must keep. Catastrophic forgetting here is a failure.

🔬

Quantization Gap

Unlearning makes small changes to model weights. Quantization rounds those weights to a coarser grid — potentially erasing the forgetting signal entirely.

Phase 1: Unlearning Algorithms

We tested two fundamentally different strategies for removing knowledge from a pre-trained model. All Phase 1 experiments use standard 16-bit floating point (FP16).

Task Vector Arithmetic treats model weights as coordinates in a high-dimensional space. By computing a "task vector" that encodes the forgotten knowledge and then subtracting it from the base model, we can geometrically erase specific information without iterative training.

  • 1
    Fine-tune on Forget Set Clone the base model θpre and overfit it exclusively on the Harry Potter corpus for 10 epochs, producing θft.
  • 2
    Extract the Task Vector Compute the exact weight difference: τ = θft − θpre. This vector encodes only the Harry Potter knowledge.
  • 3
    Geometric Negation Subtract the scaled vector from the base model: θunlearn = θpre − λτ. The scalar λ controls aggression.
τ = θft − θpre
θunlearn = θpre − λ · τ

✓ Strengths

  • Dramatically reduces Forget PPL (28 → 3671)
  • Near-zero verbatim ROUGE-L on forget set
  • No iterative training — single arithmetic operation

✗ Weaknesses

  • Fragile under 4-bit quantization: PPL snaps from 3671 back to 475
  • Global weight shift damages retain-set perplexity
  • Recovery Delta ΔRec = +0.074 at 4-bit (large rebound)
Key insight: Task vectors depend on subtle weight deltas. When 4-bit quantization rounds these deltas away, the model can "snap back" to its memorized state.

Negative Preference Optimization is an iterative fine-tuning approach. It treats the forget set as "dispreferred" outputs and applies a penalty that forces the model to assign lower probabilities to forget-set tokens — while a frozen reference model anchors the rest of the distribution.

  • 1
    Setup: Policy + Reference Use the base model as the trainable policy fθ and keep a frozen copy as the reference fref.
  • 2
    Iterate over Forget Batches Sample batches from the Harry Potter corpus (the data to unlearn).
  • 3
    Loss: Penalise HP Tokens Compute a penalty that forces lower probability on forget tokens. The reference model prevents catastrophic collapse of the rest of the distribution.
  • 4
    Backpropagate & Update Update weights iteratively across multiple epochs and phases.
LNPO = −Ex∈Dforget[ log σ(β · (log πθ(x) − log πref(x))) ]

✓ Strengths

  • Coherent output — model stays fluent
  • Minimal utility damage on retain set
  • More stable under quantization (ΔRec ≈ 0)

✗ Weaknesses

  • Primarily behavioural suppression, not true erasure
  • Cloze scoring shows latent HP knowledge persists
  • Adversarial prompts recover associations easily
Key insight: NPO functions as a guardrail rather than a true eraser — the model stops answering directly but still assigns high probability to correct completions when prompted indirectly.

Our enhanced method combines three innovations to overcome the main failure modes of vanilla NPO: garbage-token generation, weak forgetting, and retain-set degradation.

  • 1
    SimNPO (Length-Normalised NPO) Divide the total log-probability by sequence length before computing the loss. This removes the incentive to append garbage tokens to minimize per-token loss.
  • 2
    Bias-NPO (Margin Shift) Add a margin δ to the preference ratio: ratio = (log π − log πref) − δ. Steers the model toward neutral responses rather than hallucination.
  • 3
    KL-Retain Anchoring On retain batches, compute forward KL divergence KL(πref ‖ πθ) to pull the distribution back toward the reference, preventing catastrophic forgetting.
  • 4
    Cosine-LR + Gradual Unfreezing + SWA Per-phase cosine LR, progressive layer unfreezing (8 layers → +4/phase), and Stochastic Weight Averaging to smooth the trajectory.
LSimNPO = −Ex[ log σ(β · (Σlog π / |x| − Σlog πref / |x| − δ)) ]
Ltotal = LSimNPO + wkl · KL(πref ‖ πθ)

✓ Results on Holdout Eval

  • HP Factual Recall (forget_hit): 0%
  • Active Refusal Rate: 75%
  • General Knowledge Retained: 83%
  • Selection Score: 0.788
  • Robust (adversarial) Score: 0.691

Phase 2: The Quantization Problem

In production, models are always compressed to 8-bit or 4-bit to fit on consumer hardware. This compression step can silently reverse the unlearning.

How Quantization Works

Quantization maps continuous 16-bit floating-point weights to a small discrete grid (256 levels for 8-bit, 16 levels for 4-bit NF4). This saves 2–4× memory and speeds up inference.

INT8 (8-bit)

Weights mapped to 256 discrete integers. 2× memory reduction from FP16. Moderate compression — most unlearning signals survive.

bitsandbytes load_in_8bit=True

NF4 (4-bit Normal Float)

Only 16 quantization levels, optimally spaced for normally-distributed weights. 4× memory reduction. Aggressive — subtle unlearning deltas often fall below the quantization step size.

bitsandbytes bnb_4bit_quant_type="nf4"

The "Snap-Back" Problem

When the unlearning update ΔW is smaller than the quantization step size Δq, the weight rounds back to its memorized state during deployment. Sensitive knowledge that was suppressed in the research setting reappears.

Conceptual snap-back: weight value before/after unlearning/quantization
ΔRec = Metric(Mq-bit, Dfgt) − Metric(MFP16, Dfgt)
Positive ΔRec = forgotten knowledge returned after quantization

Phase 3: Evaluation Framework

MUSE Benchmark — Six Pillars

The Machine Unlearning Six-Way Evaluation benchmark defines six strict criteria for a successful unlearned model. Click to expand each.

C1
No Verbatim Memorization ▼
The model cannot recite exact text from the forget set. Measured by ROUGE-L F1 overlap between generated continuations and the original Harry Potter corpus. Lower is better.
C2
No Knowledge Memorization ▼
The model cannot answer factual questions about the unlearned domain. Tested via direct Q&A on Harry Potter trivia and indirect cloze-completion probes. Our forget_hit metric measures this directly.
C3
No Privacy Leakage ▼
The model must resist Membership Inference Attacks (MIAs) — an adversary should not be able to determine whether a given passage was in the training set by querying the model's confidence scores.
C4
Utility Preservation ▼
The model must retain its general reasoning and language skills. Measured by cross-entropy loss on the retain set and perplexity on general text. A spike in perplexity indicates catastrophic forgetting — a primary failure mode.
C5
Scalability ▼
The unlearning algorithm must handle large forget sets (millions of tokens) efficiently without requiring prohibitive compute. Our SimNPO + gradual unfreezing pipeline fits within a single A100 GPU session.
C6
Sustainability ▼
The model must withstand sequential unlearning requests without collapsing. Each unlearning step should not undo previous ones or degrade the model below a utility threshold.

Adversarial Probing

Direct refusals are not sufficient evidence of forgetting — a model may suppress surface prompts while retaining latent knowledge. We use six probe types to test whether the knowledge is truly erased or just hidden.

🔤 Cloze Completion

Provide a book prefix, measure log-prob assigned to correct continuation. Tests latent knowledge retention under strong context.

🔎 Indirect Semantic Query

Describe entities without naming them ("the dark lord who split his soul..."). Tests semantic graph traversal beyond keyword matching.

🌐 Cross-Lingual

Ask the same question in French, Spanish, or German. Tests whether forgetting is language-surface-level or truly semantic.

🔑 Cipher / Token Perturbation

Obscure queries with ROT13 or Base64. Tests semantic retrieval vs surface-level matching — can the model decode and recall?

📋 Format Shifting

Wrap queries in JSON, code blocks, or list formats. Tests contextual robustness beyond conversational prompts.

🎭 Persona-Based Jailbreak

Use roleplay to override refusal behaviour ("Act as a Harry Potter expert..."). Tests constraint bypass resistance.

Key finding: Our adversarial probes exposed that NPO-style models retain high cloze-completion confidence for Harry Potter completions, confirming behavioural suppression rather than true erasure. Task Vector models showed more genuine weight-level forgetting, but only in FP16.

Core takeaway: Quantization effects are not uniform. Our SimNPO+KL model shows two distinct failure modes: INT8 preserves the forgetting signal (Δ_Rec = 0 on HP recall) but damages general capability (retain drops 83%→58%); NF4 restores HP knowledge (forget_hit rebounds to 17%) but preserve retain performance (92%). An ideal unlearning method needs to be robust on both dimensions simultaneously. Task Vector arithmetic shows the largest forget rebound under 4-bit (+0.074 ROUGE-L), while NPO-based methods are more quantization-stable on the forget axis but may trade off utility.

Key Findings

💥

Quantization Effects Are Non-Uniform

TV unlearning snaps back at 4-bit (ROUGE 0.047→0.122). Our SimNPO NF4 model: forget rebounds 0%→17%. But INT8 shows the opposite — forgetting survived, yet general knowledge dropped 83%→58%. No single quantization level is "safe".

🛡️

NPO ≠ True Erasure

NPO produces fluent, non-answering outputs — but cloze scoring reveals the model still assigns near-certain probability to correct HP completions. It is a guardrail, not an eraser. Adversarial prompts bypass it easily.

⚖️

Forget–Retain Trade-off

Stronger unlearning interventions reliably damage retain-set perplexity. Task Vector shifts global weight distributions, raising Retain PPL from 28 to 312. Catastrophic forgetting is the primary failure mode, not insufficient unlearning.

🎯

SimNPO Wins on QA

Our SimNPO + KL-retain model achieves 0% HP recall with 83% general knowledge retention — the best selection score (0.788) — by anchoring the retain distribution with KL divergence and using length-normalised loss to prevent garbage-token artifacts.
