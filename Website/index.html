Machine Unlearning with Gemma

Eradicating targeted knowledge from LLMs while preserving general capabilities using Gradient Ascent and Task Arithmetic.

Access Models Read Findings

How It Works

📚

1. Dataset Curation

Using the muse-bench/MUSE dataset to isolate the target "forget" domain (e.g., Harry Potter) from the general "retain" domain.

📈

2. Gradient Ascent

Executing Gradient Ascent on the target corpus to reverse the weights and push the model to unlearn specific contextual associations.

🧮

3. Task Arithmetic

Calculating the Task Vector (Finetuned Weights - Pretrained Weights) and subtracting it from the base Gemma model to surgically remove knowledge.

⚡

4. 4-bit Quantization

Leveraging Unsloth and BitsAndBytes for 4-bit NormalFloat (NF4) quantization, reducing VRAM usage from 16GB to <2GB for accessibility.

Key Findings

Retain Over Forget

Traditional fine-tuning often results in catastrophic forgetting. By utilizing targeted Negative Preference Optimization (NPO) and Task Arithmetic, we successfully isolated the "forget" domain (Harry Potter) while maintaining base MMLU scores and retain-set perplexity.
